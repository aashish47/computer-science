In JavaScript, both the `new` keyword and `Object.create()` are used to create new objects and handle prototypal inheritance, but they go about it in fundamentally different ways.

Here is the short version: **`new` executes a constructor function**, while **`Object.create()` directly specifies the prototype object** for your new object without running a constructor.

---

## 1. The `new` Keyword

When you use `new`, you are invoking a **Constructor Function** (or a class). JavaScript does a lot of behind-the-scenes work here.

```javascript
function Person(name) {
	this.name = name;
}

Person.prototype.greet = function () {
	console.log(`Hi, I'm ${this.name}`);
};

const alice = new Person("Alice");
```

### What happens under the hood?

When you type `new Person('Alice')`, JavaScript:

1. Creates a brand-new, empty object.
2. Links this new object's prototype (`[[Prototype]]`) to the constructor's prototype property (`Person.prototype`).
3. Binds the `this` keyword inside the constructor to the new object.
4. Executes the constructor code (setting `this.name = 'Alice'`).
5. Returns the new object (unless the constructor explicitly returns a different object).

---

## 2. `Object.create()`

`Object.create(proto)` gives you direct, surgical control over the prototype chain. It creates a new object and sets its prototype _exactly_ to the object you pass in as the first argument.

```javascript
const personProto = {
	greet: function () {
		console.log(`Hi, I'm ${this.name}`);
	},
};

// Create a new object with personProto as its prototype
const bob = Object.create(personProto);
bob.name = "Bob"; // You have to set properties manually afterward
```

### What happens under the hood?

`Object.create()` skips the constructor step entirely. It just creates a blank object, hooks up the prototype link to `personProto`, and hands it to you. Because no constructor runs, `bob.name` doesn't exist until you explicitly define it.

---

## Key Differences At a Glance

| Feature                                  | `new Constructor()`                                                 | `Object.create(obj)`                                                                    |
| ---------------------------------------- | ------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **Primary Input**                        | A constructor function (or Class).                                  | An existing object (to be used as the prototype).                                       |
| **Constructor Execution**                | **Yes.** Runs the initialization code inside the function.          | **No.** No constructor function is invoked.                                             |
| **Property Initialization**              | Properties are set dynamically inside the constructor using `this`. | Properties must be added manually later, or via an optional second argument descriptor. |
| **Creating an Object with NO Prototype** | Not possible (it will always inherit from `Object.prototype`).      | **Possible.** `Object.create(null)` creates a completely pure dictionary object.        |

---

## When to use which?

- **Use `new` (or modern `class` syntax):** When you want a blueprint for creating _multiple_ instances of an object that all share properties and methods. It is cleaner for object-oriented programming (OOP) patterns.
- **Use `Object.create()`:** When you want to extend an existing object directly without dealing with classes, or when you explicitly need to create an object with a custom prototype (or no prototype at all via `null`).
